Getting Started

You will need to make an account to accsess this API. For almost all requests are user based and require a webtoken to be set in the headers.

Base URL

The base URL for librix is https://librix-server.herokuapp.com/.

Resources

When we dive deeper into each call it is important to know on a post request what is required. Everything that is light grey in color is not required to get a correct response.

On each request you will see a route and request type. The base url is omitted in our documnetation but will ne to be added when you make the request.

User

All user routes start with: BaseURL/user

User Schema

Add a single user

Route = / Request = POST Auth = not required

To add a single user, insert only the required fields of the user schema. The aboutMe and avatar keys can also be inserted here.

Response : An object containing the user and a token.

Get information on your match partner

Route = /users Request = POST Auth = required

Send the user id of your match partner.

Format : { id: the id of the match partner }

Response : An object containing the users general information.

Login

Route = /login Request = POST Auth = not required

To login you must send a valid email and password which must already exist within the database.

Response : An object containing the user and a token.

Logout

Route = /logout Request = GET Auth = not required

Once this route is called we will remove the any auth tokens within the request headers.

Response : A custom response object.

Add a saved book

Route = /addSavedBook Request = POST Auth = required

Send through the user id of the current user and book id of the book that user has saved.

Format : { userId: the user id, bookId: the book id }

Response : A custom response object.

Remove a saved book

Route = /removeSavedBook Request = POST Auth = required

Send through the user id of the current user and book id of the book you would like to remove from the booksToRemember array.

Format : { userId: the user id, bookId: the book id }

Response : A custom response object.

Get all your matches

Route = /(id of the current user) Request = GET Auth = required

Making this call will grab all the matches the user has made.

Response : An array containing match objects.

Update your profile

Route = /(id of the current user) Request = PUT Auth = required

Send through an object containing anything in the user schema that is not of type id.

Response : The user object.

Delete your profile

Route = /(id of the current user) Request = DELETE Auth = required

When this call is made the user will be deleted along with any books and matches the user has created.

Response : A custom response object.

Show interest in a book

Route = /(id of the current user) Request = POST Auth = required

Send through the id of the book the user is interested in, we will check for a match and send you a response in coradination with whether a match is created.

Format : { bookId: the book id }

Response : A custom response object.

Get the users library

Route = /library/(id of the current user) Request = POST Auth = required

Send through an object containing city, language and genre. If some or all of the fields are empty, the parameters for the request will be defaulted to any and no filter will be set.

Format : { city: city the user wants to trade books in, languange: language the user is interested in, genre: genre the user is interested in }

Response : An array containing book objects.

Books

All books routes start with: BaseURL/books

Book schema

Add a book

Route = / Request = POST Auth = required

To add a book, insert only the required fields of the book schema. The subtitle, description, authors,isbn, pages, personalDescription and selectedFiles keys can also be inserted here.

Response : A book object.

Get a single book

Route = /(id of the book) Request = GET Auth = required

Call this url to recieve a single book's details.

Response : A book object.

Update a book

Route = /(id of the book) Request = PUT Auth = required

Send through an object containing anything in the book schema that is not of type id.

Response : The book object.

Delete a book

Route = /(id of the book) Request = DELETE Auth = required

Call this url to delete a book object.

Response : A custom response.
